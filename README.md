##Cisco Packet Tracer – Zone-Based Policy Firewall (ZPF) Configuration
##📌 Project Overview

This project demonstrates the implementation of a Zone-Based Policy Firewall (ZPF) in Cisco Packet Tracer to secure network communication between different security zones.

The lab focuses on configuring firewall policies, traffic inspection rules, and secure communication management using Cisco IOS security features.

##🎯 Objectives
- Understand the fundamentals of Zone-Based Policy Firewall (ZPF)
- Configure security zones on Cisco routers
- Create class maps and policy maps
- Apply inspection policies between zones
- Control and filter network traffic securely
- Improve network segmentation and security


##🛠 Technologies & Tools Used
- Cisco Packet Tracer
- Cisco IOS CLI
- Zone-Based Policy Firewall (ZPF)
- Networking Fundamentals
- Access Control & Traffic Inspection



##📚 Key Concepts Covered

- Zone Pair Configuration
- Class Maps
- Policy Maps
- Traffic Inspection
- Stateful Firewall Operations
- Network Traffic Filtering
- Secure Network Design


##⚙️ Project Configuration Steps
- Configure network topology
- Assign IP addresses to devices
- Create security zones
- Configure zone pairs
- Create class maps to identify traffic
- Configure policy maps for inspection
- Apply policies to zone pairs
- Verify secure communication between zones
- Test allowed and blocked traffic scenarios


##🔐 Security Features Implemented

- Secure network segmentation
